# 🛡️ Prompt de Segurança em Camadas — Provider Mais Fibra
> **Stack:** React + Vite · TypeScript · Node.js · Drizzle ORM · PostgreSQL · Replit
> **Escopo:** Site público + Painel admin simples
> **Objetivo:** Fechar todas as portas de invasão comuns em projetos desse porte

---

## 🗺️ Mapa de Ameaças — O que este projeto atrai

| Ameaça | Vetor | Risco |
|---|---|---|
| SQL Injection | Formulários, query params | 🔴 Crítico |
| Brute Force Login | Painel admin | 🔴 Crítico |
| XSS (Cross-Site Scripting) | Campos de texto, IPTV/planos editáveis | 🔴 Crítico |
| CSRF | Formulário de contato, painel | 🔴 Crítico |
| Credential Stuffing | Login do admin | 🔴 Crítico |
| Rate Limiting / DDoS | API pública, formulário WhatsApp | 🟡 Alto |
| Path Traversal | Upload de arquivos, rotas | 🟡 Alto |
| Clickjacking | iframes embutidos | 🟡 Alto |
| Exposição de env vars | Replit, build Vite | 🟡 Alto |
| Bot / Spam | Formulário de contato | 🟡 Alto |
| Sessão sequestrada | JWT/Cookie do admin | 🟡 Alto |
| Enumeração de usuários | Mensagens de erro verbosas | 🟢 Médio |
| Dependências vulneráveis | npm packages | 🟢 Médio |

## CAMADA 1 — Banco de Dados (Drizzle ORM + PostgreSQL)

### Objetivo: Eliminar SQL Injection e limitar danos em caso de comprometimento

#### 1.1 — NUNCA usar query raw com interpolação de string

```ts
// ❌ VULNERÁVEL — SQL Injection direto
const user = await db.execute(
  `SELECT * FROM users WHERE email = '${email}'`
)

// ✅ SEGURO — Drizzle parametriza automaticamente
const user = await db
  .select()
  .from(users)
  .where(eq(users.email, email))
  .limit(1)
```

#### 1.2 — Se precisar de raw SQL, usar placeholders

```ts
// ✅ Raw SQL seguro com Drizzle
import { sql } from 'drizzle-orm'

const result = await db.execute(
  sql`SELECT * FROM planos WHERE id = ${planId} AND ativo = true`
)
// Drizzle sanitiza ${planId} — nunca concatene strings
```

#### 1.3 — Usuário do banco com permissões mínimas (Principle of Least Privilege)

```sql
-- Rodar no PostgreSQL do Replit
-- Criar usuário separado para a aplicação (não usar o superuser)
CREATE USER app_mais_fibra WITH PASSWORD 'senha_forte_aqui';

-- Conceder apenas o necessário
GRANT SELECT, INSERT, UPDATE, DELETE ON ALL TABLES IN SCHEMA public TO app_mais_fibra;
GRANT USAGE, SELECT ON ALL SEQUENCES IN SCHEMA public TO app_mais_fibra;

-- NUNCA conceder DROP, CREATE, ALTER, TRUNCATE
-- NUNCA usar o usuário postgres/root na connection string da app
```

#### 1.4 — Connection string apenas em variável de ambiente

```ts

  max: 10,                            // Limita conexões simultâneas

  connectionTimeoutMillis: 5000,

export const db = drizzle(pool)
```

## CAMADA 2 — Autenticação do Painel Admin

### Objetivo: Bloquear brute force, credential stuffing e sessões sequestradas

#### 2.1 — Instalar dependências

```bash
pnpm add bcryptjs jsonwebtoken express-rate-limit
pnpm add -D @types/bcryptjs @types/jsonwebtoken
```

#### 2.2 — Hash de senha com bcrypt (nunca MD5, SHA1 ou texto puro)

```ts
// server/utils/auth.ts
import bcrypt from 'bcryptjs'
import jwt from 'jsonwebtoken'

const SALT_ROUNDS = 12 // nunca abaixo de 10

export async function hashPassword(plaintext: string): Promise<string> {
  return bcrypt.hash(plaintext, SALT_ROUNDS)
}

export async function verifyPassword(plaintext: string, hash: string): Promise<boolean> {
  return bcrypt.compare(plaintext, hash)
}

export function generateToken(payload: { adminId: number }): string {
  if (!process.env.JWT_SECRET) throw new Error('JWT_SECRET não definido')
  return jwt.sign(payload, process.env.JWT_SECRET, {
    expiresIn: '8h',     // Expira em 8 horas
    issuer: 'mais-fibra',
    audience: 'admin-panel',
  })
}

export function verifyToken(token: string) {
  if (!process.env.JWT_SECRET) throw new Error('JWT_SECRET não definido')
  return jwt.verify(token, process.env.JWT_SECRET, {
    issuer: 'mais-fibra',
    audience: 'admin-panel',
  })
}
```

#### 2.3 — Middleware de autenticação

```ts
// server/middleware/requireAuth.ts
import { Request, Response, NextFunction } from 'express'
import { verifyToken } from '../utils/auth'

export function requireAuth(req: Request, res: Response, next: NextFunction) {
  const authHeader = req.headers.authorization
  if (!authHeader?.startsWith('Bearer ')) {
    return res.status(401).json({ error: 'Não autorizado' })
  }

  const token = authHeader.split(' ')[1]
  try {
    const decoded = verifyToken(token)
    ;(req as any).admin = decoded
    next()
  } catch {
    // Mensagem genérica — não revelar se o token expirou ou é inválido
    return res.status(401).json({ error: 'Não autorizado' })
  }
}
```

#### 2.4 — Rate limiting no login (bloqueia brute force)

```ts
// server/middleware/rateLimiter.ts
import rateLimit from 'express-rate-limit'

// Para a rota de login — muito restritivo
export const loginLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutos

  message: { error: 'Muitas tentativas. Aguarde 15 minutos.' },
  standardHeaders: true,
  legacyHeaders: false,
  skipSuccessfulRequests: true, // Não conta login bem-sucedido
})

// Para a API pública (formulário de contato, etc.)
export const apiLimiter = rateLimit({
  windowMs: 60 * 1000, // 1 minuto
  max: 30,             // 30 requisições por minuto por IP
  message: { error: 'Muitas requisições. Tente novamente em 1 minuto.' },
})

// Para o painel admin em geral
export const adminLimiter = rateLimit({
  windowMs: 60 * 1000,
  max: 100,
  message: { error: 'Limite de requisições atingido.' },
})
```

#### 2.5 — Rota de login com delay anti-timing attack

```ts
// server/routes/auth.ts
import { loginLimiter } from '../middleware/rateLimiter'
import { verifyPassword, generateToken } from '../utils/auth'

app.post('/api/auth/login', loginLimiter, async (req, res) => {
  const { email, password } = req.body

  // Sempre buscar usuário para evitar timing attack
  // (tempo de resposta idêntico para email válido e inválido)
  const admin = await db
    .select()
    .from(admins)
    .where(eq(admins.email, email?.toLowerCase()?.trim()))
    .limit(1)

  const fakeHash = '$2a$12$invalidhashtopreventtimingattack000000000000000000000'
  const hash = admin[0]?.passwordHash ?? fakeHash

  const valid = await verifyPassword(password ?? '', hash)

  if (!valid || !admin[0]) {
    // Mensagem idêntica para email inexistente e senha errada
    // Não revelar qual dos dois está errado (enumeração de usuários)
    return res.status(401).json({ error: 'Credenciais inválidas' })
  }

  if (!admin[0].ativo) {
    return res.status(403).json({ error: 'Conta desativada' })
  }

  const token = generateToken({ adminId: admin[0].id })

  // Logar tentativa bem-sucedida
  console.info(`[AUTH] Login admin ID ${admin[0].id} — IP: ${req.ip}`)

  res.json({ token, expiresIn: '8h' })
})
```

```ts

import helmet from 'helmet'

app.use(helmet({
  // Content Security Policy — define de onde podem vir scripts, estilos, imagens
  contentSecurityPolicy: {
    directives: {
      defaultSrc:     ["'self'"],
      scriptSrc:      ["'self'"],           // Sem inline scripts no admin
      styleSrc:       ["'self'", "'unsafe-inline'"], // Tailwind precisa de inline
      imgSrc:         ["'self'", "data:", "https:"],
      fontSrc:        ["'self'", "https://fonts.gstatic.com"],
      connectSrc:     ["'self'", "https://api.whatsapp.com"],
      frameSrc:       ["'none'"],           // Bloqueia iframes de terceiros
      objectSrc:      ["'none'"],
      baseUri:        ["'self'"],
      formAction:     ["'self'"],
      upgradeInsecureRequests: [],

  frameguard: { action: 'deny' },

  // Força HTTPS por 1 ano
  hsts: {
    maxAge: 31536000,
    includeSubDomains: true,
    preload: true,
  },

  // Impede sniffing de MIME type
  noSniff: true,

  // Remove header X-Powered-By (não revelar que usa Express)
  hidePoweredBy: true,

  // Referrer policy
  referrerPolicy: { policy: 'strict-origin-when-cross-origin' },

  // Bloqueia acesso a features de browser desnecessárias
  permittedCrossDomainPolicies: false,

  // Cross-Origin headers
  crossOriginEmbedderPolicy: false, // Manter false se usar imagens externas
  crossOriginOpenerPolicy: { policy: 'same-origin' },
```

```tsx
// src/utils/sanitize.ts
import DOMPurify from 'isomorphic-dompurify'

export function sanitizeHTML(dirty: string): string {
  return DOMPurify.sanitize(dirty, {
    ALLOWED_TAGS: [],    // Remove TODAS as tags HTML
    ALLOWED_ATTR: [],
  })
}

export function sanitizeText(input: string): string {
  return input
    .replace(/[<>'"]/g, c => ({ '<':'&lt;','>':'&gt;',"'":'&#39;','"':'&quot;' })[c] || c)
    .trim()
    .slice(0, 1000) // Limita tamanho máximo
}
```

```tsx
// src/hooks/useAdminApi.ts — buscar CSRF token antes de mutations
import { useEffect, useState } from 'react'

export function useAdminApi() {
  const [csrfToken, setCsrfToken] = useState('')

  useEffect(() => {
    fetch('/api/csrf-token', { credentials: 'include' })
      .then(r => r.json())
      .then(d => setCsrfToken(d.csrfToken))
  }, [])

  async function adminFetch(url: string, options: RequestInit = {}) {
    return fetch(url, {
      ...options,
      credentials: 'include',
      headers: {
        'Content-Type': 'application/json',
        'x-csrf-token': csrfToken,
        'Authorization': `Bearer ${localStorage.getItem('adminToken')}`,
        ...options.headers,
      },
    })
  }

  return { adminFetch }
}
```

```tsx
// src/components/ProtectedRoute.tsx
import { Navigate, useLocation } from 'wouter'

export function ProtectedRoute({ children }: { children: React.ReactNode }) {
  const token = localStorage.getItem('adminToken')
  const [location] = useLocation()

  if (!token) {
    // Redireciona para login sem revelar a URL tentada
    return <Navigate to="/gestao-mf/login" replace />
```

## CAMADA 6 — Proteção de Formulários Públicos (Anti-Bot/Spam)

### Objetivo: Bloquear bots no formulário de contato sem prejudicar usuários reais

#### 6.1 — Honeypot (método mais eficaz, sem impacto no UX)

```tsx
// src/components/ContatoForm.tsx
// Campo invisível para humanos, mas visível para bots
// Se preenchido = bot

function ContatoForm() {
  const [honeypot, setHoneypot] = useState('')

  async function handleSubmit(e: React.FormEvent) {
    e.preventDefault()

    // Bot preencheu o campo oculto — descartar silenciosamente
    if (honeypot) {
      // Simula sucesso para não alertar o bot
      setSubmitted(true)
      return
    }

    // Continua o envio normal...
  }

  return (
    <form onSubmit={handleSubmit}>
      {/* Campo honeypot — invisível para humanos */}
      <div style={{ position: 'absolute', left: '-9999px', opacity: 0, height: 0 }}
           aria-hidden="true">
        <label htmlFor="website">Não preencha este campo</label>
        <input
          id="website"
          name="website"
          type="text"
          tabIndex={-1}
          autoComplete="off"
          value={honeypot}
          onChange={e => setHoneypot(e.target.value)}
        />
      </div>

      {/* Campos reais do formulário */}
      {/* ... */}
    </form>
  )
}
```

```ts
export function validateEnv() {
  const missing = REQUIRED_ENV_VARS.filter(v => !process.env[v])
  if (missing.length > 0) {
    console.error(`❌ Variáveis de ambiente obrigatórias ausentes: ${missing.join(', ')}`)
    process.exit(1) // Para o servidor imediatamente
  }
  console.info('✅ Variáveis de ambiente validadas')
}

// server/index.ts — primeira linha
import { validateEnv } from './config/env'
validateEnv()
```

## CAMADA 9 — Configurações do Replit

### Objetivo: Fechar vulnerabilidades específicas do ambiente Replit

#### 9.1 — Nunca commitar secrets no Replit

```
✅ Usar: Replit Secrets (painel lateral → Secrets)
❌ Nunca usar: comentários no código, console.log de variáveis sensíveis
```

#### 9.2 — Restringir acesso ao banco pelo IP (se possível)

No PostgreSQL do Replit, a conexão já é restrita ao ambiente interno. Mas se migrar para Neon, Supabase ou outro serviço externo:
- Adicionar o IP do Replit na whitelist do banco
- Nunca deixar o banco com acesso `0.0.0.0/0` (aberto para o mundo)

#### 9.3 — Desativar modo debug em produção

```ts
// server/index.ts
if (process.env.NODE_ENV === 'production') {
  // Não expor stack traces nos erros

    res.status(500).json({ error: 'Erro interno do servidor' })
    // Nunca: res.json({ error: err.message, stack: err.stack })
  })
}
```

```ts
app.get('/health', (_req, res) => res.json({ ok: true }))
```
