## ETAPA 1 — Bundle e Code Splitting (impacto: TBT, FCP)

### Objetivo
Reduzir o JavaScript enviado na primeira carga. Uma SPA com Vite carrega tudo de uma vez por padrão — isso é o principal assassino do TBT.

### Tarefas

#### 1.1 — Configurar code splitting agressivo no Vite

```ts
// vite.config.ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  build: {
    rollupOptions: {
      output: {
        manualChunks(id) {
          // Separa vendor por biblioteca
          if (id.includes('node_modules/framer-motion')) return 'framer-motion'
          if (id.includes('node_modules/recharts')) return 'recharts'
          if (id.includes('node_modules/lucide-react')) return 'lucide'
          if (id.includes('node_modules/react-dom')) return 'react-dom'
          if (id.includes('node_modules/react')) return 'react'
          if (id.includes('node_modules')) return 'vendor'
        },
      },
    },
    chunkSizeWarningLimit: 300,
  },
})
```

#### 1.2 — Lazy load de todas as rotas (React.lazy + Suspense)

```tsx
// src/App.tsx — substituir imports estáticos por lazy
import { lazy, Suspense } from 'react'
import { Route, Switch } from 'wouter'

const Home      = lazy(() => import('./pages/Home'))
const Planos    = lazy(() => import('./pages/Planos'))
const Contato   = lazy(() => import('./pages/Contato'))
const Admin     = lazy(() => import('./pages/Admin')) // painel pesado — separa sempre

function App() {
  return (
    <Suspense fallback={<PageSkeleton />}>
      <Switch>
        <Route path="/"        component={Home} />
        <Route path="/planos"  component={Planos} />
        <Route path="/contato" component={Contato} />
        <Route path="/admin"   component={Admin} />
      </Switch>
    </Suspense>
  )
}
```

#### 1.3 — Lazy load de Framer Motion

Framer Motion é grande (~90 KB gzip). Carregue somente quando visível:

```tsx
// src/components/AnimatedSection.tsx
import { lazy, Suspense } from 'react'

const MotionDiv = lazy(() =>
  import('framer-motion').then(m => ({ default: m.motion.div }))
)

// Alternativa: usar CSS animations no above-the-fold e Framer Motion só abaixo
```

#### 1.4 — Tree-shake Lucide React

```tsx
// ❌ Errado — importa toda a biblioteca
import { Icon1, Icon2 } from 'lucide-react'

// ✅ Correto — import direto do arquivo do ícone
import Wifi from 'lucide-react/dist/esm/icons/wifi'
import Phone from 'lucide-react/dist/esm/icons/phone'
```

#### 1.5 — Analisar bundle com rollup-plugin-visualizer

```bash
pnpm add -D rollup-plugin-visualizer
```


Rode `pnpm build` e analise o relatório. Elimine dependências > 50 KB não usadas no caminho crítico.

## ETAPA 2 — Otimização de Imagens (impacto: LCP, FCP)

### Objetivo
LCP de 242 s indica que a imagem hero (ou elemento maior) não está otimizada.

### Tarefas

#### 2.1 — Converter todas as imagens para WebP/AVIF

```bash
# Instalar sharp para conversão em build
pnpm add -D sharp vite-imagetools
```

```ts
// vite.config.ts
import { imagetools } from 'vite-imagetools'
plugins: [react(), imagetools()]
```

```tsx
// Uso no componente
import heroImg from './assets/hero.jpg?format=webp&quality=80&w=1200'
```

#### 2.2 — Definir width e height em TODAS as imagens (evita CLS)

```tsx
// ❌ Sem dimensões → CLS
<img src={hero} alt="Hero" className="w-full" />

// ✅ Com dimensões → sem CLS
<img
  src={hero}
  alt="Fibra óptica de alta velocidade"
  width={1200}
  height={600}
  className="w-full h-auto"
/>
```

#### 2.3 — Preload da imagem LCP (hero)

```tsx
// src/pages/Home.tsx — dentro do <Helmet>
<Helmet>
  <link
    rel="preload"
    as="image"
    href="/images/hero.webp"
    type="image/webp"
  />
</Helmet>
```

#### 2.4 — Lazy load de imagens abaixo do fold

```tsx
// Imagens que não estão visíveis na tela inicial
<img src={img} loading="lazy" decoding="async" alt="..." />
```

#### 2.5 — Otimizar imagens no servidor (Replit)

Se as imagens vêm via API Node.js, adicionar `sharp` para servir WebP on-demand:

```ts
// server/routes/images.ts
import sharp from 'sharp'
import { Request, Response } from 'express'

app.get('/img/:filename', async (req: Request, res: Response) => {
  const accept = req.headers.accept || ''
  const format = accept.includes('image/avif') ? 'avif'
               : accept.includes('image/webp') ? 'webp'
               : 'jpeg'

  const buffer = await sharp(`./public/images/${req.params.filename}`)
    .resize({ width: 1200, withoutEnlargement: true })
    .toFormat(format, { quality: 80 })
    .toBuffer()

  res.set('Content-Type', `image/${format}`)
  res.set('Cache-Control', 'public, max-age=31536000, immutable')
  res.send(buffer)
})
```

- [ ] Todas as `<img>` têm `width` e `height` definidos
- [ ] LCP element identificado e com `preload` no `<head>`

## ETAPA 3 — Critical CSS e Render-Blocking (impacto: FCP, Speed Index)

### Objetivo
Eliminar recursos que bloqueiam a renderização inicial.

### Tarefas

#### 3.1 — Inline do CSS crítico (above-the-fold)

```ts

plugins: [
  react(),
  critical({
    criticalUrl: 'http://localhost:5173',
    criticalBase: './dist',

  }),

```

#### 3.2 — Preconnect para domínios externos

```html
<!-- index.html — adicionar no <head> -->
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<!-- Se usar analytics, CDNs, etc. -->
<link rel="dns-prefetch" href="https://www.googletagmanager.com" />
```

#### 3.3 — Carregar fontes de forma assíncrona

```html
<!-- index.html -->
<!-- ❌ Bloqueia render -->
<link href="https://fonts.googleapis.com/css2?family=Inter..." rel="stylesheet" />

<!-- ✅ Não bloqueia -->
<link rel="preload" href="https://fonts.googleapis.com/css2?family=Inter..." as="style"
      onload="this.onload=null;this.rel='stylesheet'" />
<noscript>
  <link href="https://fonts.googleapis.com/css2?family=Inter..." rel="stylesheet" />
</noscript>
```

**Melhor opção:** hospedar as fontes localmente com `fontsource`:


```ts

```

#### 3.4 — Tailwind CSS — purge agressivo

```ts

  content: [
    './index.html',
    './src/**/*.{ts,tsx}',
```

## ETAPA 4 — Cache, Compressão e Headers HTTP (impacto: Performance geral)

### Objetivo
Garantir que assets sejam cacheados e servidos comprimidos.

### Tarefas

#### 4.1 — Configurar compressão Brotli/Gzip no servidor Node.js

```bash
pnpm add compression
pnpm add -D @types/compression
```

```ts
// server/index.ts
import compression from 'compression'

app.use(compression({
  level: 9,
  threshold: 1024,
  filter: (req, res) => {
    if (req.headers['x-no-compression']) return false
    return compression.filter(req, res)
  },
}))
```

#### 4.2 — Headers de cache para assets estáticos

```ts
// server/index.ts
import { Request, Response, NextFunction } from 'express'

// Assets com hash no nome (gerados pelo Vite) — cache longo
app.use('/assets', (req: Request, res: Response, next: NextFunction) => {
  res.set('Cache-Control', 'public, max-age=31536000, immutable')
  next()
}, express.static('./dist/assets'))

// HTML — sem cache (sempre revalida)
app.use(express.static('./dist', {
  setHeaders: (res, path) => {
    if (path.endsWith('.html')) {
      res.set('Cache-Control', 'no-cache, no-store, must-revalidate')
    }
  },
}))
```

#### 4.3 — Vite: garantir hashes nos nomes dos arquivos de build

```ts
// vite.config.ts
build: {
  assetsDir: 'assets',
  rollupOptions: {
    output: {
      entryFileNames:   'assets/[name]-[hash].js',
      chunkFileNames:   'assets/[name]-[hash].js',
      assetFileNames:   'assets/[name]-[hash][extname]',
      // manualChunks: ... (definido na Etapa 1)
    },
  },
}
```

#### 4.4 — Habilitar HTTP/2 no Replit (se disponível)

No Replit, verificar se o proxy suporta HTTP/2. Se sim, nenhuma configuração adicional é necessária. Se usar domínio customizado, configurar via Cloudflare (próxima etapa).

#### 6.7 — Animações: respeitar prefers-reduced-motion

```tsx
// src/hooks/useReducedMotion.ts
import { useEffect, useState } from 'react'

export function useReducedMotion() {
  const [reduced, setReduced] = useState(
    window.matchMedia('(prefers-reduced-motion: reduce)').matches
  )
  useEffect(() => {
    const mq = window.matchMedia('(prefers-reduced-motion: reduce)')
    const handler = (e: MediaQueryListEvent) => setReduced(e.matches)
    mq.addEventListener('change', handler)
    return () => mq.removeEventListener('change', handler)
  }, [])
  return reduced
}

// Uso no componente
const reducedMotion = useReducedMotion()

<motion.div
  initial={{ opacity: 0, y: reducedMotion ? 0 : 20 }}
  animate={{ opacity: 1, y: 0 }}
  transition={{ duration: reducedMotion ? 0 : 0.4 }}
>
```

#### 7.2 — Sitemap XML + robots.txt no servidor Node.js

```ts
// server/routes/seo.ts
import { Router } from 'express'
const router = Router()

const PAGES = ['/', '/planos', '/contato', '/sobre']
const BASE_URL = 'https://seu-dominio.com.br'

router.get('/sitemap.xml', (req, res) => {
  const urls = PAGES.map(p => `
  <url>
    <loc>${BASE_URL}${p}</loc>
    <changefreq>weekly</changefreq>
    <priority>${p === '/' ? '1.0' : '0.8'}</priority>
  </url>`).join('')

  res.set('Content-Type', 'application/xml')
  res.send(`<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
${urls}
</urlset>`)
})

router.get('/robots.txt', (req, res) => {
  res.type('text/plain')
  res.send(`User-agent: *
Allow: /
Sitemap: ${BASE_URL}/sitemap.xml`)
})

export default router
```

```tsx
export function ThirdPartyScripts() {
  useEffect(() => {
    // Carregar após interação ou 5s, o que vier primeiro
    const load = () => {
      const script = document.createElement('script')
      script.src = 'https://www.googletagmanager.com/gtag/js?id=GA_ID'
      script.async = true
      document.head.appendChild(script)
    }

    const timer = setTimeout(load, 5000)
    const events = ['click', 'scroll', 'keydown', 'mousemove']
    const onInteraction = () => { clearTimeout(timer); load(); events.forEach(e => window.removeEventListener(e, onInteraction)) }
    events.forEach(e => window.addEventListener(e, onInteraction, { once: true }))

    return () => { clearTimeout(timer); events.forEach(e => window.removeEventListener(e, onInteraction)) }
  }, [])

  return null
}
```

#### 10.1 — Wake-up do servidor (cold start)

```ts
app.get('/health', (req, res) => res.status(200).json({ ok: true }))
```

```ts
// server/index.ts — garantir que o build é servido corretamente
import path from 'path'
import { fileURLToPath } from 'url'

const __dirname = path.dirname(fileURLToPath(import.meta.url))
const distPath = path.resolve(__dirname, '../dist') // ajustar conforme estrutura monorepo

app.use(express.static(distPath, {
  maxAge: '1y',
  etag: true,
  lastModified: true,
}))

// SPA fallback — todas as rotas retornam index.html
app.get('*', (req, res) => {
  res.sendFile(path.join(distPath, 'index.html'))
})
```
