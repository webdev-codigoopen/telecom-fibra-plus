# Responsive Design Skill

Skill para criar interfaces **verdadeiramente responsivas**, com foco em excelente usabilidade mobile e adaptação fluida para todos os dispositivos.

---

## Filosofia Central

**Mobile-First**: Comece sempre pelo menor dispositivo e expanda.  
**Content-First**: O conteúdo guia o layout, não o contrário.  
**Touch-First**: Interactions desenhadas para dedos, não cursores.  
**Performance-First**: Mobile tem menos CPU, RAM e bandwidth — otimize.

## Breakpoints Padrão

```css
/* ─── SISTEMA DE BREAKPOINTS ─── */

/* Mobile pequeno (320px – 480px) — iPhone SE, Android compact */
/* Base styles: sem media query — mobile-first */

/* Mobile grande (481px – 767px) — iPhone Plus, Android XL */
@media (min-width: 481px) { }

/* Tablet (768px – 1024px) — iPad, Android tablet */
@media (min-width: 768px) { }

/* Notebook 13–14" (1025px – 1366px) — MacBook Air/Pro 13", Dell XPS 13 */
@media (min-width: 1025px) { }

/* Notebook 15" (1367px – 1600px) — MacBook Pro 15/16", Dell XPS 15 */
@media (min-width: 1367px) { }

/* Desktop largo (1601px+) — monitores externos, iMac */
@media (min-width: 1601px) { }

/* ─── DETECÇÃO ESPECIAL ─── */

/* Alta densidade de pixel (Retina) */
@media (-webkit-min-device-pixel-ratio: 2), (min-resolution: 192dpi) { }

/* Preferência por menos animação */
@media (prefers-reduced-motion: reduce) { }

/* Modo escuro do sistema */
@media (prefers-color-scheme: dark) { }

/* Landscape em mobile */
@media (max-width: 767px) and (orientation: landscape) { }
```

## Regras Obrigatórias — Mobile

### 1. Meta Viewport
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover">
```
`viewport-fit=cover` é essencial para iPhone com notch (safe areas).

### 2. Safe Areas (iPhone Notch / Dynamic Island)
```css
:root {
  --safe-top: env(safe-area-inset-top);
  --safe-bottom: env(safe-area-inset-bottom);
  --safe-left: env(safe-area-inset-left);
  --safe-right: env(safe-area-inset-right);
}

.header { padding-top: calc(1rem + env(safe-area-inset-top)); }
.bottom-nav { padding-bottom: calc(0.75rem + env(safe-area-inset-bottom)); }
```

### 3. Touch Targets — NUNCA menores que 44×44px
```css
/* Regra Apple HIG e Google Material — alvos mínimos de toque */
button, a, [role="button"], input[type="checkbox"], input[type="radio"] {
  min-height: 44px;
  min-width: 44px;
  /* Para elementos visualmente menores, use padding para expandir a área de toque */
}

/* Ícones pequenos: área de toque maior que o visual */
.icon-btn {
  width: 24px;
  height: 24px;
  padding: 10px; /* área de toque real: 44×44px */
}
```

### 4. Tipografia Fluida
```css
:root {
  /* Escala fluida: min no mobile, max no desktop */
  --text-xs:   clamp(0.75rem,  1.5vw, 0.875rem);
  --text-sm:   clamp(0.875rem, 2vw,   1rem);
  --text-base: clamp(1rem,     2.5vw, 1.125rem);
  --text-lg:   clamp(1.125rem, 3vw,   1.25rem);
  --text-xl:   clamp(1.25rem,  3.5vw, 1.5rem);
  --text-2xl:  clamp(1.5rem,   4vw,   2rem);
  --text-3xl:  clamp(1.75rem,  5vw,   2.5rem);
  --text-4xl:  clamp(2rem,     6vw,   3.5rem);
}

/* Nunca use font-size < 16px em inputs — evita zoom automático no iOS */
input, textarea, select { font-size: max(16px, 1rem); }

/* Line-height confortável para mobile */
body { line-height: 1.6; }
```

### 5. Scroll e Overflow
```css
/* Scroll suave nativo em iOS */
html { scroll-behavior: smooth; -webkit-overflow-scrolling: touch; }

/* Evitar scroll horizontal indesejado */
body { overflow-x: hidden; max-width: 100vw; }

/* Scroll horizontal controlado (carrosséis, tabelas) */
.scroll-x {
  overflow-x: auto;
  -webkit-overflow-scrolling: touch;
  scroll-snap-type: x mandatory;
  scrollbar-width: none; /* Firefox */
}
.scroll-x::-webkit-scrollbar { display: none; }

.scroll-x > * { scroll-snap-align: start; }
```

## Layout por Dispositivo

### Mobile (320px – 767px)
```css
/* Layout base — mobile first */
.container {
  width: 100%;
  padding: 0 1rem; /* 16px laterais */
  max-width: 480px;
  margin: 0 auto;
}

/* Grid: sempre 1 coluna no mobile */
.grid { 
  display: grid;
  grid-template-columns: 1fr;
  gap: 1rem;
}

/* Cards: full-width */
.card { width: 100%; }

/* Navegação: bottom bar (padrão nativo mobile) */
.nav-mobile {
  position: fixed;
  bottom: 0;
  left: 0; right: 0;
  height: 56px;
  padding-bottom: env(safe-area-inset-bottom);
  display: flex;
  align-items: center;
  justify-content: space-around;
  background: white;
  border-top: 1px solid #e5e7eb;
  z-index: 100;
}
```

### Tablet (768px – 1024px)
```css
@media (min-width: 768px) {
  .container {
    padding: 0 2rem;
    max-width: 1024px;
  }

  /* 2 colunas */
  .grid { grid-template-columns: repeat(2, 1fr); gap: 1.5rem; }

  /* Sidebar + conteúdo */
  .layout-sidebar {
    display: grid;
    grid-template-columns: 240px 1fr;
    gap: 2rem;
  }

  /* Navegação: sidebar ou top bar */
  .nav-mobile { display: none; }
  .nav-sidebar { display: flex; }
}
```

### Notebook 13–14" (1025px – 1366px)
```css
@media (min-width: 1025px) {
  .container { padding: 0 2.5rem; max-width: 1200px; }

  /* 3 colunas */
  .grid { grid-template-columns: repeat(3, 1fr); gap: 2rem; }

  .layout-sidebar {
    grid-template-columns: 260px 1fr;
  }
}
```

### Notebook 15" (1367px – 1600px)
```css
@media (min-width: 1367px) {
  .container { padding: 0 3rem; max-width: 1440px; }

  /* 4 colunas possível */
  .grid-4 { grid-template-columns: repeat(4, 1fr); }

  .layout-sidebar { grid-template-columns: 280px 1fr; }
}
```

### Desktop largo (1601px+)
```css
@media (min-width: 1601px) {
  .container { max-width: 1600px; padding: 0 4rem; }
  .grid-4 { grid-template-columns: repeat(4, 1fr); gap: 2.5rem; }
}
```

## Navegação Responsiva — Padrões

### Bottom Navigation (Mobile nativo)
```html
<nav class="bottom-nav">
  <a href="/" class="nav-item active">
    <svg><!-- ícone --></svg>
    <span>Home</span>
  </a>
  <!-- itens -->
</nav>
```
```css
.bottom-nav {
  position: fixed; bottom: 0; left: 0; right: 0;
  display: flex;
  padding-bottom: env(safe-area-inset-bottom);
  background: white;
  box-shadow: 0 -1px 0 rgba(0,0,0,0.1);
}
.nav-item {
  flex: 1;
  display: flex; flex-direction: column; align-items: center;
  padding: 0.5rem;
  min-height: 56px;
  gap: 4px;
  font-size: 0.75rem;
  color: #6b7280;
  text-decoration: none;
}
.nav-item.active { color: #2563eb; }

@media (min-width: 768px) { .bottom-nav { display: none; } }
```

### Hamburger → Drawer (Tablet/Desktop alternativo)
```css
.hamburger { display: flex; } /* mobile */

@media (min-width: 768px) {
  .hamburger { display: none; }
  .nav-links { display: flex; } /* links horizontais */
}
```

## Imagens e Mídia Responsiva

```css
/* Regra universal */
img, video, svg, canvas {
  max-width: 100%;
  height: auto;
  display: block;
}

/* Hero image com proporção fixa */
.hero-image {
  aspect-ratio: 16/9;
  object-fit: cover;
  width: 100%;
}

/* Avatar responsivo */
.avatar {
  width: clamp(40px, 10vw, 80px);
  height: clamp(40px, 10vw, 80px);
  border-radius: 50%;
}
```

```html
<!-- Imagens com srcset para diferentes densidades -->
<img 
  src="hero-800.jpg"
  srcset="hero-400.jpg 400w, hero-800.jpg 800w, hero-1200.jpg 1200w"
  sizes="(max-width: 480px) 100vw, (max-width: 1024px) 90vw, 1200px"
  alt="Descrição da imagem"
  loading="lazy"
  decoding="async"
>

<!-- Picture para formatos modernos -->
<picture>
  <source srcset="hero.avif" type="image/avif">
  <source srcset="hero.webp" type="image/webp">
  <img src="hero.jpg" alt="...">
</picture>
```

## Forms — Usabilidade Mobile

```html
<!-- Tipos corretos de input ativam teclado correto no mobile -->
<input type="email"     inputmode="email"   autocomplete="email">
<input type="tel"       inputmode="tel"     autocomplete="tel">
<input type="number"    inputmode="numeric" pattern="[0-9]*">
<input type="search"    inputmode="search">
<input type="url"       inputmode="url">
<input type="text"      inputmode="text"    autocomplete="name">
```

```css
/* Campos de formulário mobile-friendly */
input, textarea, select {
  width: 100%;
  min-height: 48px; /* acima do mínimo de 44px */
  padding: 0.75rem 1rem;
  font-size: max(16px, 1rem); /* evita zoom no iOS */
  border-radius: 0.5rem;
  border: 1.5px solid #d1d5db;
  appearance: none; /* remove estilos nativos inconsistentes */
  -webkit-appearance: none;
}

/* Labels acima do campo — padrão mais legível no mobile */
label {
  display: block;
  font-size: 0.875rem;
  font-weight: 500;
  margin-bottom: 0.375rem;
}

/* Botão de submit: full-width no mobile */
.btn-submit {
  width: 100%;
  min-height: 48px;
  font-size: 1rem;
  font-weight: 600;
}

@media (min-width: 768px) {
  .btn-submit { width: auto; padding: 0.75rem 2rem; }
}
```

## Performance Mobile

### CSS Crítico
```css
/* Prefira transform/opacity para animações — aceleração por GPU */
.animate {
  transform: translateY(0);
  opacity: 1;
  transition: transform 0.3s ease, opacity 0.3s ease;
  will-change: transform, opacity; /* hint para o browser */
}

/* Evite: top, left, width, height em animações (causam reflow) */
```

### Evitar Reflows
```css
/* Use aspect-ratio para reservar espaço de imagens */
.img-wrapper {
  aspect-ratio: 16/9;
  background: #f3f4f6; /* placeholder enquanto carrega */
}

/* Skeleton loading */
.skeleton {
  background: linear-gradient(90deg, #f3f4f6 25%, #e5e7eb 50%, #f3f4f6 75%);
  background-size: 200% 100%;
  animation: shimmer 1.5s infinite;
}
@keyframes shimmer {
  0% { background-position: 200% 0; }
  100% { background-position: -200% 0; }
}
```

### Font Loading
```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>
```
```css
/* Font display swap — texto visível imediatamente */
@font-face {
  font-family: 'MyFont';
  src: url('font.woff2') format('woff2');
  font-display: swap;
}
```

## Gestos e Interações Touch

```css
/* Cursor pointer em elementos interativos */
button, a, [role="button"] { cursor: pointer; }

/* Remover highlight azul/cinza no toque (iOS/Android) */
* { -webkit-tap-highlight-color: transparent; }

/* Desabilitar seleção de texto em elementos interativos */
button, nav, .card { user-select: none; -webkit-user-select: none; }

/* Feedback de toque — estado :active visível */
.btn:active { transform: scale(0.97); transition: transform 0.1s; }
.card:active { opacity: 0.9; }

/* Overscroll behavior */
.modal-scroll {
  overscroll-behavior: contain; /* evita scroll do body quando modal está aberto */
}
```

## Acessibilidade Mobile

```css
/* Focus visível para teclados físicos (Bluetooth) e navegação por tab */
:focus-visible {
  outline: 2px solid #2563eb;
  outline-offset: 2px;
  border-radius: 4px;
}

/* Remover outline apenas para mouse (não teclado) */
:focus:not(:focus-visible) { outline: none; }
```

```html
<!-- ARIA labels em ícones sem texto -->
<button aria-label="Fechar menu">
  <svg aria-hidden="true"><!-- ícone X --></svg>
</button>

<!-- Skip link para acessibilidade -->
<a href="#main-content" class="skip-link">Ir para o conteúdo</a>
```
```css
.skip-link {
  position: absolute;
  top: -100%;
  left: 1rem;
  background: #2563eb;
  color: white;
  padding: 0.5rem 1rem;
  border-radius: 0 0 0.5rem 0.5rem;
  z-index: 1000;
}
.skip-link:focus { top: 0; }
```

## Tailwind CSS — Classes Responsivas

```html
<!-- Mobile-first com prefixos sm: md: lg: xl: 2xl: -->
<div class="
  w-full                    /* mobile */
  md:max-w-2xl              /* tablet */
  lg:max-w-4xl              /* notebook */
  xl:max-w-6xl              /* desktop */
  mx-auto
  px-4 md:px-8 lg:px-12
">

<!-- Grid responsivo -->
<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-4 lg:gap-6">

<!-- Texto responsivo -->
<h1 class="text-2xl sm:text-3xl md:text-4xl lg:text-5xl font-bold">

<!-- Navegação responsiva -->
<nav class="
  fixed bottom-0 left-0 right-0   /* mobile: bottom nav */
  md:static md:flex               /* tablet+: horizontal */
">
```

**Breakpoints Tailwind mapeados:**
| Prefixo | Largura | Dispositivo |
|---------|---------|-------------|
| (base)  | 0px+    | Mobile      |
| `sm:`   | 640px+  | Mobile grande |
| `md:`   | 768px+  | Tablet      |
| `lg:`   | 1024px+ | Notebook 13" |
| `xl:`   | 1280px+ | Notebook 15" |
| `2xl:`  | 1536px+ | Desktop     |

Para notebook 15" específico, use CSS customizado ou estenda o Tailwind:
```js
// tailwind.config.js
screens: { 'notebook-15': '1367px' }
```

## Checklist de Qualidade

Antes de entregar qualquer interface, verifique:

**Layout**
- [ ] Sem scroll horizontal não intencional
- [ ] Container com `max-width` e `padding` laterais em todos os breakpoints
- [ ] Grid colapsa corretamente: 4col → 3col → 2col → 1col

**Mobile**
- [ ] Touch targets ≥ 44×44px
- [ ] Font-size de inputs ≥ 16px (evita zoom iOS)
- [ ] Safe areas aplicadas (notch/dynamic island)
- [ ] Bottom navigation para mobile quando há múltiplas seções

**Tipografia**
- [ ] `clamp()` para títulos principais
- [ ] `font-display: swap` para web fonts
- [ ] `line-height` ≥ 1.5 para corpo de texto

**Imagens**
- [ ] `max-width: 100%` em todas as imagens
- [ ] `loading="lazy"` em imagens abaixo da dobra
- [ ] `aspect-ratio` para evitar layout shift

**Performance**
- [ ] Animações usando `transform` e `opacity` apenas
- [ ] `will-change` apenas onde necessário
- [ ] `prefers-reduced-motion` respeitado

**Acessibilidade**
- [ ] `:focus-visible` visível
- [ ] `aria-label` em botões de ícone
- [ ] Contraste de cores adequado (WCAG AA: 4.5:1)
